# NeKI — Newsletter Knowledge Index

NeKI is a compact index to selected third-party developer writing. Entries are leads for research and implementation ideas; they are not Apple documentation, API contracts, or approved architecture decisions. NeKI publishes routing metadata, not the linked articles' text.

## Lookup

- [Sources](sources.md) — publications currently represented by NeKI.
- [Newsletters](newsletters/) — curated editions with selected external reading where anonymous full-text access was verified.
- [Blogs](blogs/) — anonymously readable entries from each source's article or long-form catalogue.

For an individual publication's indexed entries, use [newsletters](newsletters/) for curated editions or [blogs](blogs/) for direct article catalogues. Every listed newsletter issue links to its original page and may include date, topics, visible sections, and all anonymously readable editorial links that passed NeKI's selection rules. A concise NeKI-authored brief appears only when the page provides enough context for a useful original abstract. Topic files separate direct-source reading from newsletter leads and list every deduplicated matching entry without top-N sampling.
